# 수익 모델 & 구독 전략

---

## 현재 수익 구조

- **티켓 시스템**: 1티켓 = 450원 (인앱결제, 토스페이먼츠 경유)
- **무료 획득**: 데일리 퀘스트(러닝기록+쿨다운 → 1티켓), 친구 초대(30티켓)
- **유료**: 인앱결제 → RevenueCat → 티켓 지급

## 페이서스 2.0 구독 모델 (2026-06-22 확정)

### 핵심 결정
1. **구독 + 티켓 공존** (폐지 아님)
2. **구독**: 월 5,500~7,800원 (가격 미확정) → 모든 유료기능 무제한
3. **티켓 인상**: 호감보내기 3→5~6티켓으로 인상 → 자주 쓰는 유저 구독 유도
4. **신규 유료 기능 2개**: 오늘의 카드(사주/타로) + 마라톤 훈련 플랜 → 구독 트리거

### 구독 포함 기능
- 같이 달려요 (1:1 호감)
- 데일리크루 호감
- 오늘의 카드 (운세 — ⏸️ HOLD)
- 마라톤 훈련 플랜

### "행운의 7%" 캠페인
구독료(앱스토어 수수료 제외)의 7% → 포캣멍센터 후원

### 구현 방식
- BDK 기반 구독 모델 (Natively 이주 보류 2026-06-23)
- 가우라브 BDK 구독 템플릿 대기 중
- RevenueCat에 구독 Offering/Entitlement 신규 추가 필요

### 미완성 작업
- [ ] 정확한 구독 가격 확정
- [ ] 정확한 티켓 인상값 확정
- [ ] RevenueCat 구독 SKU 생성
- [ ] User 테이블 `is_subscriber`(yes/no) 필드 추가
- [ ] 모든 티켓 차감 워크플로우 앞단에 "구독자면 차감 스킵" 분기 추가
- [ ] RevenueCat webhook에 구독 이벤트(`RENEWAL`/`CANCELLATION`/`EXPIRATION`) 핸들링 추가
- [ ] 티켓 가격 인상 공지 계획
