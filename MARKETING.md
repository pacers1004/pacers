## 0. 페이서스 2.0 — 구독모델 "행운의 7%" (2026-06-22 확정)

> 신규 유료기능(오늘의 카드, 마라톤 플랜) 출시를 계기로 1,000명 시점에 구독모델을 바로 도입. 2,000명까지 기다리지 않는다 — 매출 적은 지금이 BM 전환 리스크가 가장 작은 타이밍.

### 핵심 구조
- **신규 유료기능 2개**: 오늘의 카드(사주/타로 운세, 티켓 소모) + 마라톤 훈련 플랜(기존 §06, 버블 연동 미완)
- **기존 유료기능 유지**: 같이 달려요/호감보내기, 데일리크루 등 — 모두 티켓 차감 유지 (티켓 모델 폐지 아님, 구독과 공존)
- **신규 월 구독**: 5,500원~7,800원 (가격 미확정) — 구독자는 위 모든 유료기능 무제한 무료
- **티켓 가격 인상**: 단가(450원)는 유지, 호감보내기 소모량 3→5~6티켓 인상 (1회 1,350원→2,250~2,700원) — 자주 쓰는 유저일수록 구독이 유리해지도록 설계. 3~4회만 보내도 구독료보다 비싸짐(뽕 뽑는 구조)
- **하이브리드 유지 이유**: 저빈도 유저 보호 + 티켓이 구독가 비교 기준점 역할(앵커링) + freemium/subscription 하이브리드는 업계 표준 패턴

### 구독 전용 신규 혜택
1. **호감보내기 메시지 첨부** (우선순위 높음) — 현재 원터치 호감전송에 메시지 입력 추가, 매칭 성사율(핵심 KPI) 직접 개선 기대. 신고/차단 플로우 동반 설계 필요
2. **파워러너 뱃지** — 기존 Badge 데이터타입 활용, 구독자 전용 프리미엄 뱃지를 데일리카드 프로필에 노출 → 신뢰 신호
3. **구독자 전용 러닝 인사이트** — 매칭 성사율/페이스 트렌드 등 데이터 심화 분석
4. **오픈런/데일리크루 우선 신청권** — 인기 코스 마감 전 우선 신청 (단, 매칭 우선노출은 비구독자 차별 인상 우려로 제외)

### 기부 연계 — "행운의 7%" 캠페인
- 구독료(앱스토어 수수료 제외)의 **7%**를 포캣멍센터(유기동물 보호단체) 후원
- **이미 검증된 자산** — 과거 "1km당 1,000원 매칭 기부" 챌린지 실시 + 포캣멍센터 500,000원 기부 확인증(2026-01-12) + 앱 내 매거진으로 투명 공유 이력 보유. 신규 개발 불필요, 기존 자산 확장
- 네이밍 "행운의 7%" — 구독자(러너)와 수혜동물 양쪽에 행운을 비는 의미, "럭키 세븐" 보편적 긍정 연상 활용 (단순 숫자보다 캐치프레이즈가 바이럴에 유리 — 미네워터·탐스 등 코즈마케팅 성공사례 공통점)
- 학술 근거: warm-glow giving 효과 실재(Nielsen: 소비자 56%가 사회적 가치 기업에 더 지불 의사) / 단, 기부액 과다 시 회의감 유발 → 7%는 적정 수준
- 확정 문구: "구독료(앱스토어 수수료 제외)의 7%는 포캣멍센터 후원에 사용됩니다 — 행운의 7%, 당신과 강아지 모두에게 행운이 있기를. 이후 기부 비율은 사정에 따라 조정될 수 있습니다."
- 집행: 분기 또는 연말 일괄 (절대금액 작아 매월 처리는 비효율), 매거진으로 투명 공개

### 실행 순서 (Gemini 의견 검토 후 합의)
1. Natively 결제 배선 먼저 완료 (RevenueCat webhook 등 — 구독을 얹을 인프라 기반, §27·§29 참고)
2. 오늘의 카드 / 마라톤 플랜 베타 빠르게 출시 (완벽하지 않아도 됨, 1,000명을 클로즈드 베타로 활용)
3. 구독 모델을 그 직후 묶어서 출시
4. 가격 프로모션은 "첫 달 100원"이 아니라 7일 무료체험 또는 첫 달 50% 할인 (100원은 가격 앵커가 너무 낮아져 정상가 전환 시 이탈 위험)

### 추가 행동경제학적 기대 효과
- 매몰비용/flat-rate bias: 구독자는 "무제한인데 안 쓰면 손해"라는 압박으로 같이 달려요를 더 자주 사용할 가능성 — 현재 낮은 매칭 밀도 문제를 반대 방향으로 해결하는 레버

### B. 유튜브 자동화 로드맵 (2026-06-22 수립)

> 카드뉴스 영상 형태(정적 캐러셀→영상 변환)와 "진짜 영상"(AI 생성 B롤/모션그래픽) 두 트랙을 단계적으로 자동화한다.

**1단계 (지금, 비용 0원) — 수동 업로드 + 제목 SEO만 표준화**
- 대표님이 캡컷에서 만든 릴스 영상을 유튜브에도 동시 업로드
- 제목/태그는 매번 Claude가 SEO 템플릿(§2-B 키워드 우선순위)으로 작성해서 전달
- 목적: 자동화 없이 메타데이터 개선 효과부터 검증

**2단계 (반자동화) — Dropbox/Drive → Make → YouTube 업로드**
- 대표님이 완성 영상을 지정 폴더에 넣기만 하면 끝
- Make.com 흐름: `Dropbox(새 파일 감지) → YouTube(Upload a Video) → 제목/설명은 HTTP 모듈로 Claude API 호출해 카드뉴스 후킹 문구를 SEO 제목으로 변환`
- Make.com YouTube 모듈 지원 확인됨 (영상 업로드, 썸네일 업로드, 채널 배너 설정, 임의 API 호출 가능)
- 전제: Google 계정 인증을 페이서스 채널 소유 계정으로 연결해야 함 (현재 다른 계정으로 연결되어 있을 가능성 있음 — 확인 필요)

**3단계 (완전자동화, 장기) — "움직이는 카드뉴스" 멀티플랫폼 동시 배포**
- Placid Video API로 카드뉴스 5장을 모션그래픽 영상(텍스트 슬라이드인 + 러닝 스톡 풋티지 배경)으로 자동 생성
- 동일 산출물을 Make.com이 인스타그램 + 유튜브에 동시 업로드
- 이 단계가 구축되면 유튜브 자동화는 인스타 자동화에 무임승차

**보안/운영 주의**: YouTube OAuth 연결 시 반드시 페이서스 채널 관리 권한이 있는 구글 계정으로 연결할 것. 잘못된 계정 연결 시 엉뚱한 채널에 업로드/수정될 위험.

## 4-1. AI 영상 생성 도구 활용 전략 (Higgsfield / Kling / Midjourney)

> **결론**: "카드뉴스 영상" 너머의 "진짜 영상"(시네마틱 B롤, 무드샷)은 AI로 제작 가능하지만, **AI 생성 영상을 메인 콘텐츠로 쓰면 안 된다.** 2026년 기준 소비자의 83%가 AI 생성 영상을 식별할 수 있고, 신뢰 리스크가 크다. 유튜브도 "대량생산·반복적 콘텐츠"에 대한 수익화 단속을 강화하는 중이다.
> **원칙**: AI는 B롤·배경·트랜지션 등 보조 역할만. 브랜드의 "얼굴"과 "진짜 사람"이 메인이 되어야 신뢰와 전환이 따라온다.

### 도구별 비교 및 적용처

### 실전 적용 가이드
1. **메인 콘텐츠(후킹 카드뉴스, CTA)는 AI 영상 금지** — 카피/디자인 그대로 유지, Placid Video API로 모션그래픽화하는 정도까지만.
2. **B롤/무드샷에만 AI 사용** — 새벽 도시, 신발끈, 발 클로즈업 등 얼굴 없는 장면. Higgsfield 또는 Kling으로 생성해서 카드뉴스 영상의 배경/인트로에 합성.
3. **진짜 사람 콘텐츠가 최우선 ROI** — 대표님(또는 실제 유저) 얼굴이 나오는 15초 클립은 AI로 대체 불가. 월 2~3개라도 직접 촬영해서 끼워 넣는 게 AI 영상 전체보다 전환 효과가 크다.
4. **비용 우선순위**: 지금 단계에선 Higgsfield Starter($15/월)나 Kling API(건당 과금, $9.80부터)로 가볍게 테스트 — 월 구독보다 사용한 만큼 내는 API 방식이 초기 검증에 안전.
